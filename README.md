# Potential-driven biological flow in a synthetic 3D conduit

A compact C++17 **reference transport demonstration** for geometry- and potential-driven dynamics in a synthetic three-dimensional biological conduit-like domain.

This repository examines how transport dynamics in living systems can be represented through the coupling between spatial organization, driving fields, local conductance, boundary conditions, and time-dependent state variables. The reference implementation uses a synthetic three-dimensional conduit-like domain to make this coupling explicit: geometry defines the domain, a scalar potential field defines the driving gradient, conductance modulates transport, and a dynamic state variable evolves through time.

The implementation provides a reproducible scientific-computing workflow: define a domain, define a driving potential, simulate transport, export scientific outputs, and visualize the resulting dynamics. Its value is not tied to a specific biological system or programming language. Instead, it makes explicit the formal objects that must be specified before a computational model can be connected to measurements, perturbations, validation, and mechanistic interpretation.

## Scientific motivation

Mechanistic modeling becomes scientifically useful when the relevant variables, assumptions, and relations are visible.

A model should not be treated as a closed computational package whose inputs and outputs are analyzed only after calibration. Its scientific value depends on whether its geometry, state variables, rates, boundary conditions, parameters, uncertainties, and outputs can be inspected, modified, reproduced, and connected to measurement.

This repository follows that principle. A synthetic 3D conduit-like structure is generated computationally. A scalar pressure/potential field defines a driving force. A transported marker evolves through the domain. The resulting dynamics are rendered as a rotating 3D animation and exported through reproducible outputs.

The computational workflow links:

- computational geometry;
- potential gradients;
- transport direction;
- diffusion or dispersion;
- source--sink forcing;
- temporal dynamics;
- scientific visualization;
- reproducible code and outputs.

The same formal architecture can be adapted to different biological dynamics when the forces, scales, parameters, observables, and boundary conditions are redefined for each system.

## Conceptual background

Previous work on the Runoff Potential Index developed the idea that a 3D spatial structure can be transformed into interpretable dynamic information by combining local curvature, gradient magnitude, and divergence-like reasoning.

In that context, terrain morphology was not treated as static geometry only. Geometry was used to infer redistribution potential across 3D surfaces. This repository translates the same broader modeling attitude into a different computational setting: a synthetic biological conduit where geometry and potential gradients jointly organize transport.

The connection is conceptual rather than application-specific:

- in the Runoff Potential Index, surface morphology informs water redistribution potential;
- in this reference transport demonstration, conduit-like morphology and a pressure/potential field inform transport dynamics;
- in both cases, the aim is to move from static structure toward interpretable process.

This makes the repository a bridge between geometry-derived dynamic indices, mechanistic biological modeling, and scientific visualization.

## Mathematical model

The reference implementation represents potential-driven transport in a synthetic 3D conduit-like domain.

A scalar potential field $P(\mathbf{x})$ defines a driving field. The induced transport direction is approximated as:

$$
\mathbf{u}(\mathbf{x}) = -K(\mathbf{x}) \nabla P(\mathbf{x})
$$

where:

- $P(\mathbf{x})$ is a pressure-, water-potential-, chemical-, or mechanical-potential-like field;
- $K(\mathbf{x})$ represents an effective conductance or permeability;
- $\mathbf{u}(\mathbf{x})$ is the induced transport field.

A transported marker $c(\mathbf{x},t)$ evolves according to a simplified advection--diffusion formulation:

$$
\frac{\partial c}{\partial t} + \nabla \cdot \left(c\mathbf{u}\right) = D\nabla^2 c + S
$$

where:

- $c(\mathbf{x},t)$ is a concentration-like state variable;
- $D$ is an effective diffusion or dispersion coefficient;
- $S$ represents source terms, inlet forcing, or localized supply.

The formulation is transparent: geometry, fields, parameters, forcing, and outputs are declared explicitly so that the model can be inspected, reproduced, modified, and connected to measurements in future system-specific applications.

## Biological transferability

The computational structure can be reinterpreted across different living systems by changing geometry, forcing, parameterization, boundary conditions, and measurement relations.

| Modeling component | Role | Possible biological interpretation |
|---|---|---|
| 3D conduit geometry | Spatial constraint | Vessel, duct, root-associated channel, polar growth domain, tissue-like conduit |
| Potential field $P$ | Driving force | Pressure, suction, water potential, chemical potential, mechanical gradient |
| Conductance $K$ | Local transport capacity | Permeability, resistance, tissue conductance, hydraulic capacity |
| Transported marker $c$ | Dynamic state variable | Tracer, solute, signal, transported material, concentration-like quantity |
| Source/sink $S$ | Boundary forcing | Inlet forcing, uptake, local demand, growth-associated source or sink |

For pressure-driven biological systems, the potential field can represent a simplified pressure-like driver.  
For root or plant-associated systems, it can represent suction, water potential, or uptake-driven transport.  
For polarized growth or morphogenesis-inspired systems, it can represent a localized potential or directional field guiding material redistribution.

The transferable element is the formal architecture, not automatic biological validity. Scientific interpretation requires system-specific measurements, units, perturbations, and validation evidence.

## Computational geometry and visualization

The implementation includes a lightweight computational-graphics component.

The C++ code generates rotating frames that visualize:

- the conduit-like spatial domain;
- the transported state variable;
- the propagation of concentration-like material through time;
- the effect of potential-driven transport.

The animation is generated from the simulated fields rather than from a pre-rendered external model. Visualization is therefore part of the reproducible computational workflow.

```text
C++ geometry generation
→ potential field definition
→ transport simulation
→ scientific output
→ frame rendering
→ GIF assembly
```

Python is used only to assemble the generated frames into a GIF.

## Scope

This reference implementation is a formal and computational demonstration. It is designed to make the modeling chain explicit: geometry, field, transport rule, state variable, forcing, output, and visualization.

Biological interpretation requires additional system-specific evidence, including measured geometry, potential gradients, flux or velocity fields, state-variable dynamics, perturbation responses, parameterization, uncertainty assessment, and validation under controlled experimental conditions.

## Build

```bash
mkdir -p build
cd build
cmake ..
cmake --build .
```

## Run

From the `build/` directory:

```bash
./potential_flow_demo ../output
```

Expected outputs include:

```text
output/frames/frame_0000.ppm
output/frames/frame_0001.ppm
...
output/vtk/
```

## Generate the GIF

From the repository root:

```bash
python3 scripts/make_gif.py output/frames docs/demo_preview_v1_2.gif
```

Alternatively, using ImageMagick:

```bash
magick -delay 4 -loop 0 output/frames/frame_*.ppm docs/demo_preview_v1_2.gif
```
